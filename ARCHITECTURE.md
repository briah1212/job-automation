## Data Flow: Complete Vertical Slice

### 1. User Registration & Profile Creation
```
User вҶ’ Frontend вҶ’ POST /api/auth/register вҶ’ Backend
                                          вҶ’ Hash password
                                          вҶ’ Create user record
                                          вҶ’ Return JWT token

User вҶ’ Frontend вҶ’ POST /api/profile вҶ’ Backend
                                    вҶ’ Create profile
                                    вҶ’ Store work history, skills
```

### 2. Resume Upload
```
User вҶ’ Frontend вҶ’ POST /api/resumes вҶ’ Backend
                                    вҶ’ Upload file to MinIO
                                    вҶ’ Calculate hash
                                    вҶ’ Parse PDF (mock parser)
                                    вҶ’ Extract structured data
                                    вҶ’ Create ResumeFamily + ResumeVersion
                                    вҶ’ Status: needs_review
```

### 3. Job Import
```
User вҶ’ Frontend вҶ’ POST /api/jobs/import-url вҶ’ Backend
                                            вҶ’ Fetch URL
                                            вҶ’ Create WorkflowTask
                                            вҶ’ Queue for processing

Background Worker:
  вҶ’ ExtractionAgent.extract(raw_html)
     вҶ’ AI Gateway (Mock Provider)
     вҶ’ Return ExtractedJob schema
  вҶ’ ClassificationAgent.classify(job_data)
     вҶ’ Return JobClassification
  вҶ’ Save CanonicalJob
  вҶ’ Status: scored

  вҶ’ MatchingAgent.score(job, profile)
     вҶ’ Calculate dimension scores
     вҶ’ Return MatchScore with explanation
  вҶ’ Update job.score
```

### 4. Application Preparation
```
User вҶ’ Frontend вҶ’ POST /api/applications вҶ’ Backend
                                         вҶ’ Create Application record
                                         вҶ’ Create WorkflowTask

Background Worker:
  вҶ’ ResumeSelectionAgent.select(job, resumes, profile)
     вҶ’ Return selected_resume_id + rationale
  
  вҶ’ TailoringAgent.tailor(job, resume, profile)
     вҶ’ Generate requirement-evidence matrix
     вҶ’ Reorder/rewrite bullets (preserve facts)
     вҶ’ Return ResumeTailoring with provenance
  
  вҶ’ QuestionAgent.answer(questions, profile)
     вҶ’ Classify risk level
     вҶ’ Generate answers from verified facts
     вҶ’ Return answers with sources
  
  вҶ’ ReviewAgent.review(application, resume, answers)
     вҶ’ Independent verification
     вҶ’ Check for contradictions
     вҶ’ Check claim provenance
     вҶ’ Return ReviewResult
  
  вҶ’ Update Application status: awaiting_review
```

### 5. User Review & Approval
```
User вҶ’ Frontend вҶ’ GET /api/applications/{id}
                вҶ’ Display:
                   вҖў Resume diff viewer
                   вҖў Generated answers
                   вҖў Risk classifications
                   вҖў Review findings
                
User approves вҶ’ POST /api/applications/{id}/approve
             вҶ’ Update status: approved
```

### 6. Browser Submission
```
User вҶ’ Frontend вҶ’ POST /api/applications/{id}/start-browser

Backend вҶ’ Create WorkflowTask
       вҶ’ Queue for browser-worker

Browser Worker:
  1. process_application(application_id)
  2. Get application data from API
  3. Launch Playwright browser
  4. Navigate to job URL
  5. Detect ATS type (check data-ats attribute)
  6. Select adapter (MockATSAdapter)
  7. inspect_form()
     вҶ’ Extract all fields with labels
     вҶ’ Detect required fields
     вҶ’ Map to canonical names
  8. For each field:
     вҶ’ Get answer from application data
     вҶ’ fill_field(field, answer)
  9. upload_document(resume_file)
  10. Create checkpoint
  11. Pause (Assisted Mode)
      вҶ’ Status: paused
      вҶ’ Waiting for user confirmation

User reviews preview вҶ’ POST /api/applications/{id}/resume
                    вҶ’ Continue workflow

Browser Worker:
  12. Resume from checkpoint
  13. preview_submission()
  14. submit()
  15. detect_confirmation()
      вҶ’ Look for success indicators
      вҶ’ Extract application ID
  16. Take final screenshot
  17. Update Application status: confirmed
  18. Create AuditEvent
```

## Security Model

### Authentication Flow
1. User submits credentials
2. Backend validates with bcrypt
3. Generate JWT with user_id claim
4. Return token (expires in 24h)
5. Frontend stores in HTTP-only cookie
6. All API calls include Authorization header
7. Backend validates token on each request
8. Dependency injection provides current_user

### Authorization Rules
- Users can only access their own data
- All API routes check user_id from JWT
- Database queries filter by user_id
- File URLs are pre-signed (expire in 1 hour)

### Sensitive Data Handling
- Passwords hashed with bcrypt
- JWT secrets in environment variables
- Sensitive profile fields in separate table (future)
- AI calls redact SSN, passwords, etc.
- Browser screenshots redact sensitive fields
- Audit logs exclude full sensitive values

### Browser Security
- No CAPTCHA circumvention
- Pause on authentication required
- Restricted to approved domains
- No arbitrary code execution
- Screenshot retention configurable
- User can disable automation per site
